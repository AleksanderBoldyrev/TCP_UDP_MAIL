1) Сервер
- основной поток обработки ui и запуска-остановки сервера. Для остановки сервера - будет отдельная глобальная переменная.
- tcp server - слушает и принимает соединения
- udp server - слушает и принимает соединения
- класс обработчик - запускается для каждого клиента свой экземпляр класса, обрабатывает команды
- каждый обработчик имеет свой экземпляр класса работы с файлами
- файловый обработчик - работает с файлом, уникальным для каждого пользователя
- глобальный массив булевых переменных, где количество значений - это количество пользователей. Если пользователь залогинен - то значение 1, иначе 0.
- глобальный массив пользователей
- идентефикация пользователя производится по имени и паролю. Имя - подпапка папки с данными, пароль лежит внутри в файле ###

2) Клиент
- tcp client
- udp client
- обработчик api

3) api
- логин (клиент -> сервер)
	- да (серве - клиент)
	- нет (сервер - клиент)
- отправить письмо пользователю (клиент - сервер)
	- отправилось (сервер - клиент)
	- не отправилось (сервер - клиент)
- проверить ящик (клиент - сервер)
	- есть информация (сервер - клиент)
	- ошибка (сервер - клиент)
- получение кокретного письма по номеру (клиент - сервер)
	- есть письмо (сервер - клиент)
	- нет письма (сервер - клиент)
- разорвать соединение (клиент - сервер)
- разорвать соединение (сервер - клиент)

4) База: 
Для каждого пользователя создаем папку в папке Data, где будут храниться его письма в виде файлов с номерами и файл с паролем с именем "###"
